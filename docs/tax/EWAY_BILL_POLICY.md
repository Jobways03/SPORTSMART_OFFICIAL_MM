# EWAY_BILL_POLICY.md

**Purpose:** E-way bill generation, validation, and ship-blocking behaviour.

---

## 1. When required

Per CBIC Rule 138, an e-way bill is required for the movement of goods of consignment value exceeding **₹50,000** (single national default). Some states have higher intra-state thresholds (e.g. ₹1,00,000 in some cases).

Determination at order time:
```
if (subOrder.invoiceTotalInPaise > tax_config.eway_bill_threshold_for(fromState, toState, transportMode))
  eway_bills.status = REQUIRED
else
  eway_bills.status = NOT_REQUIRED
```

`tax_config.eway_bill_thresholds` is a JSON config — engineering seeds defaults; CA confirms per-state overrides.

## 2. Generation timing

- Required **before dispatch**.
- Triggered when seller marks the sub-order `PACKED` (i.e. just before `SHIPPED`).
- If seller attempts to mark `SHIPPED` while `eway_bills.status = REQUIRED`:
  - **Strict mode:** ship action refused with error "E-way bill required for this consignment."
  - **Test mode:** warning logged; ship proceeds (NOT production-safe).

## 3. Adapter modes

### 3.1 Stub (current — `EWAY_BILL_PROVIDER=stub`)

- Generates placeholder EWB number `EWB-STUB-{uuid}`.
- Logs the request payload to `eway_bills.rawPayloadJson` so dev can see what would have been sent.
- `validUntil` set per simple km-slab table (≤100 km = 1 day, +200 km/day capped at 15 days).
- Allows development testing of the ship-block, customer-visible state, admin retry UI without NIC creds.

### 3.2 NIC e-Waybill API (`EWAY_BILL_PROVIDER=nic` — not yet wired)

- Real NIC e-Waybill v1.04 API.
- Requires Sportsmart's GST username + password (per-GSTIN credentials).
- Token-based auth (token cached for ~6h).
- Auto-generates IRN if integrated with e-invoicing.

**CA must confirm** when Sportsmart wants to integrate NIC EWB API — typically tied to e-invoicing decision.

## 4. Fields captured

| Field | Source |
|---|---|
| EWB number | Provider response (or stub UUID) |
| EWB date | Provider response (or generation time) |
| Valid until | Provider response (or km-slab computation) |
| Distance (km) | Computed: Haversine between dispatch warehouse pincode and shipping pincode (uses existing `PostOffice` table) |
| Vehicle number | Optional — entered by seller at ship-confirmation |
| Transporter ID | Optional — `tax_config.default_transporter_id` (Sportsmart's GST transporter ID, if any) |
| Transport mode | ROAD by default; admin/seller selectable (ROAD / RAIL / AIR / SHIP) |
| From pincode + state | Dispatch warehouse address (from `SellerProductMapping.warehousePincode` or `Seller.sellerZipCode`) |
| To pincode + state | Shipping address |
| Consignment value | Invoice total (post-discount, including GST and shipping) |
| Invoice ref | Linked `tax_documents.id + documentNumber` |

## 5. Validity & expiry

EWBs expire on `validUntil`. If expired before delivery:
- Customer-visible state: unchanged (no leakage).
- Admin AdminTask `EWAY_BILL_EXPIRED` raised if shipment hasn't reached destination.
- Seller can request EWB extension via admin (NIC API supports up to 8h extension).
- After 8h grace, a new EWB must be generated (carrier change typically required).

## 6. Failure handling

Provider call failures:
- Captured in `eway_bills.failureReason + rawPayloadJson`.
- AdminTask `EWAY_BILL_GENERATION_FAILED` raised.
- Retry cron `eway_bill_retry_processor` attempts every 30 min, up to 6 retries.
- After exhaustion: seller cannot ship; finance/ops manual intervention required.

## 7. Cancellation

- EWBs can be cancelled within 24 hours of generation (NIC rule).
- Order cancellation post-EWB: triggers `EwayBillService.cancel(ewbId)`.
- Status flips `GENERATED → CANCELLED`.
- New EWB issued if same consignment ships under new circumstances.

## 8. Multi-package consignments

For a single sub-order shipping in multiple packages (rare):
- Single EWB per **sub-order**, not per package.
- All packages travel under the same EWB number.
- Vehicle number captures only the primary vehicle (per CBIC convention).

## 9. PDF

E-way bill PDF includes:
- EWB number prominently
- QR code (placeholder until NIC integration)
- All fields from §4
- "This is a computer-generated document" footer

Generated by `EwayBillPdfService`, stored at `tax-documents/EWAY_BILL/{fy}/{sellerGstin}/{ewbNumber}.pdf`.

## 10. Permissions

- `tax.ewayBill.read` — view EWBs on order/admin pages
- `tax.ewayBill.generate` — trigger generation (also implicit on `PACKED` transition)
- `tax.ewayBill.cancel` — cancel EWB
- `tax.ewayBill.override` — allow ship without EWB (admin only; audited; CA-monitorable)
