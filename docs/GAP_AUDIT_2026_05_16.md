# SportSmart Marketplace — Deep Gap & Bug Audit

**Audit Date:** 2026-05-16
**Lead:** Engineering team review
**Method:** 6 parallel deep-dive sweeps across the full monorepo, verified against current code (not memory).
**Scope:** NestJS API (42 modules) + 8 Next.js frontends + Prisma schema + infra + CI.
**Comparison baseline:** 2026-05-14 audit (project memory). Items confirmed STILL OPEN, RESOLVED, or NEW are tagged below where relevant.

---

## How to Read This Document

Every row in every table follows the same shape:

| Column | Meaning |
|---|---|
| **Severity** | `CRIT` (ship-blocker / money loss / compliance break) → `HIGH` (data integrity / security) → `MED` (UX / minor leaks) → `LOW` (polish). |
| **Type** | `BUG` (broken thing), `GAP` (missing feature), `RACE` (concurrency hole), `SECURITY`, `PERF`, `A11Y`, `SEO`, `PII`. |
| **Title** | One-line summary. |
| **File:Line** | Exact location (relative to `apps/api/src/` or `apps/web-*/src/` unless noted). |
| **Impact** | What breaks for the user/business in plain English. |

Numbering convention: `§<section>.<subsection>` (e.g., §3.5 = Settlements). Issue IDs `[D<section>-<n>]` for cross-referencing in tickets.

## Executive Summary

| Severity | Count | One-line characterization |
|---|---|---|
| **CRITICAL** | **11** | Ship-blockers — broken core flows, money-loss risk, XSS in email, compliance failures |
| **HIGH** | **53** | Auth bypass, integration races, missing crons, missing UI safety nets |
| **MEDIUM** | **74** | UX bugs, schema drift, plain-text fallbacks, soft-fail patterns |
| **LOW** | **28** | Polish, performance, future-proofing |
| **TOTAL** | **166** | (after de-dup with 2026-05-14 audit) |

### Top 11 CRITICAL (in attack order)

1. **Inventory write use-cases are stubs** with obfuscated `Uxxxxx` class names (UsetUvariantUstock, UreserveUstock, UdeductUstock, UreleaseUstock) — `inventory/application/use-cases/*`. Variant stock writes do nothing. [D4-11]
2. **Email HTML injection (XSS)** across order, return, dispute, support email templates — customer name / seller name / product title interpolated unescaped. [D6-1, D6-2, D6-9]
3. **Seller onboarding ~80% unimplemented** — `submit-seller-onboarding`, `approve-seller`, `reject-seller` all throw `Not implemented`. [D1-2]
4. **Multi-payment refund split not implemented** — entire `refund-instructions` module. Refunds for orders paid via wallet+card all route to one method. [D3-4]
5. **Wallet ledger recon cron disabled by default** — `WALLET_LEDGER_RECON_ENABLED=false`. Wallet drift silent for months. [D3-15]
6. **Search reindex is a no-op stub** — `search/use-cases/rebuild-search-index.use-case.ts`. Catalog changes never reach OpenSearch. [D4-19]
7. **`PERMISSIONS_GUARD_STRICT=false` by default** — 403s never thrown in prod; permission checks log-only. [D1-7]
8. **Audit module write use-cases stubbed** — `write-audit-log.use-case.ts` + `write-event-log.use-case.ts` both `Not implemented`. Chain anchor cron runs but use-case path is dead code. [D2-13]
9. **Hardcoded `admin@sportsmart.com`** in return notification handler (2 places). [D5-9]
10. **`Dockerfile.api` has placeholder digest `sha256:0000…`** — first prod build fails or pulls unpinned upstream. [D9-1]
11. **No graceful SIGTERM HTTP draining** in `apps/api/src/main.ts` — pod evictions abort in-flight payments. [D10-2]

### Recommended attack order (compressed from agent recs)

1. Fix CRITICAL #1–#3 (Inventory stubs, Email XSS, Seller onboarding) — required for basic operation.
2. Fix CRITICAL #4–#6 (Refund split, Wallet recon, Search reindex) — required for money & catalog correctness.
3. Flip `PERMISSIONS_GUARD_STRICT=true` and `IDEMPOTENCY_ENABLED=true` in staging.
4. Wire audit use-cases (#8) + remove hardcoded admin email (#9).
5. Pin Dockerfile digest (#10) + add SIGTERM (#11) + enable reconciliation crons.
6. Address all HIGH items domain-by-domain (Sections 3 → 6 are densest).
7. Frontend safety net pass: `error.tsx`/`not-found.tsx` across 7 of 8 apps.
8. Storefront SEO/Analytics (sitemap, robots, generateMetadata, GA4, cookie consent).
9. Schema integrity fixes (SKU unique, soft-delete normalize, enum drift).
10. Observability (OpenTelemetry, business metrics, expanded health checks).
11. DPDP final pass (data-export endpoint, Cloudinary cleanup, backup/PITR docs).
