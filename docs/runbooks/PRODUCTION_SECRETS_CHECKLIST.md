# Production Secrets Checklist — fill `production/app/external`

The one manual secrets step before production go-live. Everything here goes into a
**single** AWS Secrets Manager secret named **`production/app/external`**, in region
**Mumbai (ap-south-1)**, account **943189351633**.

> **You only paste these 14 operator keys.** The database URL, Redis URL, JWT
> secrets, and `ADMIN_MFA_ENCRYPTION_KEY` are auto-generated by Terraform into the
> separate `production/app/generated` secret — **do not add those here.** If you
> don't see them, that's expected; they're created for you.
>
> **These don't exist until production is provisioned.** Terraform creates
> `production/app/external` (empty/placeholder) during the first
> `terraform apply -var-file=production.tfvars`; you fill it afterward. Terraform
> will **never overwrite** what you paste (the value is `ignore_changes`).
>
> **Production refuses to boot on placeholders.** Under `NODE_ENV=production` the
> API/shipping service crash-loop until the required values below are real — this
> is the intended safety lock (see [PRODUCTION_APEX_CUTOVER.md](./PRODUCTION_APEX_CUTOVER.md)
> and [ENVIRONMENT.md](../ENVIRONMENT.md)).

## 💳 Razorpay — payments
**Required to boot.** The API will not start in production without these three.

| Field | Where to get it |
|---|---|
| `RAZORPAY_KEY_ID` | Razorpay Dashboard → **Settings → API Keys** (in **Live** mode, KYC-approved). Starts `rzp_live_…` |
| `RAZORPAY_KEY_SECRET` | Shown **once** when you generate the live key — copy it immediately |
| `RAZORPAY_WEBHOOK_SECRET` | Razorpay Dashboard → **Settings → Webhooks** → create a webhook to the API's webhook URL → the secret **you set** there |

## 📦 Cloudflare R2 — file / image storage
**Required to boot** (first 4). `R2_PUBLIC_BASE_URL` is required for image uploads to work (missing ⇒ uploads 502).

| Field | Where to get it |
|---|---|
| `R2_ACCOUNT_ID` | Cloudflare → **R2** → Account ID (also in the R2 endpoint URL) |
| `R2_BUCKET` | the bucket **name** you created in R2 |
| `R2_ACCESS_KEY_ID` | Cloudflare R2 → **Manage R2 API Tokens → Create** → Access Key ID |
| `R2_SECRET_ACCESS_KEY` | the Secret shown with that same token (copy it then — shown once) |
| `R2_PUBLIC_BASE_URL` | the public URL images are served from — the bucket's `…r2.dev` URL **or** a custom domain (e.g. `https://cdn.sportsmart.com`) |

## ✉️ Email — OTP / notification mail
Without these, the API runs "log-only" mail (OTPs and notifications are **not actually sent**).

| Field | Where to get it |
|---|---|
| `MAIL_USER` | your SMTP provider username (SES SMTP user, SendGrid, Gmail app login, etc.) |
| `MAIL_PASS` | the matching SMTP password / app password |

## 🚚 Delhivery — courier / shipping
**Required to boot the shipping service** in production (it crash-loops on placeholder values).

| Field | Where to get it |
|---|---|
| `DELHIVERY_API_URL` | **`https://track.delhivery.com`** (production host — confirm on one.delhivery.com) |
| `DELHIVERY_API_TOKEN` | Delhivery **One** panel → API setup → token |
| `DELHIVERY_CLIENT_NAME` | your registered Delhivery client / warehouse name |
| `DELHIVERY_WEBHOOK_TOKEN` | the shared secret you set for Delhivery webhooks |

## How to paste them
1. AWS Console → **Secrets Manager** → region **Mumbai (ap-south-1)** → open **`production/app/external`**.
2. **Retrieve secret value → Edit** — you'll see the field names above with empty/placeholder values.
3. Fill each value → **Save**. (Tip: open `staging/app/external` in another tab to mirror the exact field names.)
4. **Redeploy / restart the API and the shipping service** so they re-read the secret (it's read only at startup):
   ```
   gh workflow run deploy.yml -f target=production -f services=all
   ```

CLI alternative to set all values at once (build the JSON, then):
```
aws secretsmanager put-secret-value \
  --secret-id production/app/external \
  --secret-string file://prod-external.json \
  --region ap-south-1
```

## Order
Do **Razorpay (3) + R2 (4)** first — the hard "won't boot" keys — then `R2_PUBLIC_BASE_URL`, email, and Delhivery (needed before the shipping service comes up). Then redeploy and confirm the API health page is green:
```
curl -fsS https://api.sportsmart.com/api/v1/health/ready    # expect HTTP 200
```

## Verify what's set (no values printed)
```
aws secretsmanager get-secret-value --secret-id production/app/external \
  --region ap-south-1 --query SecretString --output text | jq 'to_entries | map({(.key): (.value|length>0)}) | add'
```
Shows each field as `true` (set) / `false` (still empty) without revealing the values.
